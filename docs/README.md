# neko_no_te ドキュメント

## サマリ

このドキュメントは、neko_no_te プロジェクトの概要、セットアップ手順、および基本的な使用方法を提供します。

## イントロダクション

neko_no_te は、AIアシスタント機能を提供するRustベースのプロジェクトです。

## プロジェクト構成

このリポジトリはRustで構築されたプロジェクトであり、アプリケーション本体と複数のプラグインで構成されています。
以下に、プロジェクトの基本的な構成と開発手順を示します。

### フォルダ構成

- `neko-assistant/`: 対話型AIアシスタント機能を提供するGUIアプリケーション
- `plugins/`: アシスタントの拡張機能を提供するプラグイン群
- `docs/`: プロジェクトのドキュメントが格納されているディレクトリ
- `tech_demo/`: 技術的検証用のコードが格納されているディレクトリ

### MCP サーバー管理 UI

`neko-assistant` のチャットウィンドウには「Manage MCP」ボタンがあり、Model Context Protocol サーバーの登録・編集・削除をすべて GUI 上で行えます。フォームに入力した内容は実行バイナリと同じディレクトリ（`cargo run` 時は `target/debug/mcp_servers.json`）に即座に保存されるため、JSON を直接編集する必要はありません。スクリーンショットは現在準備中ですが、`cargo run -p neko-assistant` で簡単に確認できます。

### docsフォルダ構成

詳細なドキュメントは角クレート毎に `docs/` ディレクトリを作成し、そこに配置します。
ルートの `docs/` フォルダにはプロジェクト全体に関するドキュメントを配置します。
各

- `README.md`: プロジェクトの概要とセットアップ手順を記載したファイル
- `architecture/`: プロジェクトのアーキテクチャに関するドキュメント - 詳細は各フォルダ内で記述されるのでおおざっぱな説明のみ
- `design/`: プロジェクトの設計に関するドキュメント
- `usage/`: ユーザー向けの使用方法に関するドキュメント
- `development/`: 開発者向けのドキュメント
- `api/`: API仕様に関するドキュメント
- `examples/`: 使用例やチュートリアルに関するドキュメント
- `faq/`: よくある質問とその回答に関するドキュメント
- `changelog/`: 変更履歴に関するドキュメント
